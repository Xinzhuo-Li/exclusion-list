Usage Guide

Web Search

Use the Search page to query exclusion records. Two filters are available:

  • Name — partial match on first name, middle name, last name, or business name
  • NPI — exact 10-digit National Provider Identifier

Reinstated providers are included in search results by default. Source state, list type, and date filters are not available on the search form; use the result table columns to review those fields after a match.

REST API

Programmatic access is available at /api/v1/. Interactive documentation:

Open API Documentation

Search parameters

Only q (or name) and npi are supported for filtering exclusions. Other query parameters are ignored.

Example Requests

# Search by NPI
curl "{{ base_url }}/api/v1/exclusions/?npi=1234567890"

# Search by name (matches first, middle, last, or business name)
curl "{{ base_url }}/api/v1/exclusions/?q=Smith"

# API alias for name
curl "{{ base_url }}/api/v1/exclusions/?name=Action+Pharmacy"

# List data sources
curl "{{ base_url }}/api/v1/sources/"

# Summary statistics
curl "{{ base_url }}/api/v1/stats/"

Pagination: responses include count, next, and previous. Default page size is 50; use ?page_size=100 (max 200).

Data Updates

Data is refreshed when the ETL pipeline runs and merges new state files into exclusion_main. The web application reads live from the database — no redeployment is needed after a data update.
